Название прецедента: change user password.

Действующее лицо: Пользователь.

Цель: изменить пароль пользователя.

Предусловия: пользователь вошел в систему.

Главная последовательность:
  Пользователь выбирает в главном меню пункт "Изменить имя, пароль".
  Система выводит окно изменения имени, пароля.
  Поле для ввода имени недоступно для редактирования, и отображает текущее имя пользователя.
  Пользователь меняет пароль в поле для ввода нового пароля.
  Пользователь подтверждает новый пароль в поле для проверки пароля.
  Пользователь нажимает кнопку "Изменить".
  Система проверяет правильность введенного пароля.
  Система система изменяет пароль пользователя.
  Система делает запись в журнале логирования.
  Система выводит на экран сообщение об изменении пароля.
  Пользователь нажимает кнопку "Да" в сообщении.
  Система закрывает окна сообщения и изменения имени, пароля.
  Система возвращает фокус в главное окно программы.
  
Альтернативная последовательность: пользователь нажимает кнопку "Отмена".
  Система закрывает окно изменения имени, пароля.
  Система возвращает фокус в главное окно программы.
    
Альтернативная последовательность: пользователь ввёл слишком короткий пароль.
  Система выводит предупреждающее сообщение.
  Пользователь нажимает кнопку "Да" в сообщении.
  Система закрывает предупреждающее окно.
  Система возвращает фокус в окно изменения имени, пароля.
  
Альтернативная последовательность: пароли в полях "Новый пароль" и "Проверка пароля" не совпадают.
  Система выводит предупреждающее сообщение.
  Пользователь нажимает кнопку "Да" в сообщении.
  Система закрывает предупреждающее окно.
  Система возвращает фокус в окно изменения имени, пароля.
