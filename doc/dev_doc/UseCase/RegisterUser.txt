Название прецедента: Register new admin and user.

Действующее лицо: Администратор.

Цель: регистрация нового пользователя/администраратора в системе.

Предусловия: администратор вошёл в систему.

Главная последовательность: 
  Администратор выбирает в Главном меню пункт "Добавить пользователя".
  Система показывает окно добавления пользователя в систему.
  Администратор вводит имя в поле "Имя".
  Администрато вводит пароль в поле "Пароль".
  Администратор подтверждает пароль в поле "Подтвердить пароль".
  Администратор выбирает категорию пользователя (user/admin).
  Администратор нажимает кнопку "Добавить".
  Система проверяет правильность введённых данных.
  Система добавляет нового пользователя.
  Система делает запись в журнале логирования.
  Система выводит на экран окно с сообщением о добавлении нового пользователя.
  Администратор нажимает кнопку "Да" в сообщении.
  Система закрывает окна сообщения и добавления нового пользователя.
  Система возвращает фокус в главное окно приложения.
  
Альтернативная последовательность: администратор нажимает кнопку "Отмена".
  Система закрывает окно добавления пользователя в систему.
  Система возвращает фокус в главное окно программы.
    
Альтернативная последовательность: администратор не заполнил поле с именем пользователя.
  Система выводит окно с предупреждающим сообщнием.
  Администратор нажимает кнопку "Да" в сообщении.
  Система закрывает окно сообщения.
  Система возвращает фокус в окно добавления нового пользователя.
  
Альтернативная последовательность: администратор ввёл слишком короткий пароль.
  Система выводит окно с предупреждающим сообщнием.
  Администратор нажимает кнопку "Да" в сообщении.
  Система закрывает окно сообщения.
  Система возвращает фокус в окно добавления нового пользователя.
  
Альтернативная последовательность: пароли в полях "Пароль" и "Подтвердить пароль" не совпадают.
  Система выводит окно с предупреждающим сообщнием.
  Администратор нажимает кнопку "Да" в сообщении.
  Система закрывает окно сообщения.
  Система возвращает фокус в окно добавления нового пользователя.
  
Альтернативная последовательность: имя пользователя занято.
  Система выводит окно с предупреждающим сообщнием.
  Администратор нажимает кнопку "Да" в сообщении.
  Система закрывает окно сообщения.
  Система возвращает фокус в окно добавления нового пользователя.
